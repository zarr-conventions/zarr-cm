# Track upstream v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Snapshot the newly-published upstream `v0.1` of
spatial/proj/multiscales as new package revisions (spatial r3, proj r3,
multiscales r1+r2 via flat→package conversion), wire multiscales into the drift
checker, and refresh examples.

**Architecture:** spatial/proj get a new frozen `_r3.py` snapshot pinned to the
v0.1 commit (spatial: URL-only change; proj: relaxed `proj:code` pattern +
`anyOf` CRS rule). multiscales converts from a flat module to a revision package
(`_r1` verbatim with dangling-v1 URLs, `_r2` at v0.1) mirroring spatial/proj,
with its `_contract.py` check migrating to per-revision. Three v0.1 schemas are
vendored; the drift checker tracks all three against the v0.1 snapshots.

**Tech Stack:** Python 3.11+, mypy strict (hook scoped to `src` — run
`uv run pre-commit run mypy --files <paths>`), pytest (pinned `>=9,<9.1`), ruff,
pylint (10.00/10 via `uvx nox -s pylint`). Tests NOT a package (absolute
imports). DO NOT use `git commit --no-verify`. If a commit aborts because a hook
reformatted, `git add` again and re-commit.

**Reference spec:** `docs/superpowers/specs/2026-06-16-track-v0.1-design.md`

```python
def validate(data: dict[str, Any]) -> GeoProjAttrs:
    """Validate proj (r3) data.

    At least one of ``proj:code``, ``proj:wkt2``, or ``proj:projjson`` must be
    present, and ``proj:code`` (if present) must match ``^[^:]+:[^:]+$``.
    """
    present = [k for k in ("proj:code", "proj:wkt2", "proj:projjson") if k in data]
    if not present:
        msg = (
            "At least one of 'proj:code', 'proj:wkt2', 'proj:projjson' must be present"
        )
        raise ValueError(msg)
    if "proj:code" in data and not _CODE_PATTERN.match(data["proj:code"]):
        msg = f"'proj:code' must match {_CODE_PATTERN.pattern!r}, got {data['proj:code']!r}"
        raise ValueError(msg)
    return data  # type: ignore[return-value]
```
